# Using This Project in Xcode

## ‚úÖ Yes, you can use this project in Xcode!

This is an Expo project, which means you need to generate the native iOS project first.

## Step 1: Generate Native iOS Project

Run this command to generate the `ios/` folder:

```bash
npx expo prebuild --platform ios
```

This will:
- Create the `ios/` folder with Xcode project files
- Configure all Expo plugins
- Set up native dependencies

## Step 2: Open in Xcode

After prebuild completes:

```bash
open ios/*.xcworkspace
```

Or manually:
1. Navigate to `ios/` folder
2. Open the `.xcworkspace` file (NOT `.xcodeproj`)

## Step 3: Configure Your Project

### Required Setup:

1. **Signing & Capabilities**
   - Select your app target
   - Go to "Signing & Capabilities"
   - Add your Apple Developer Team
   - Configure Bundle Identifier

2. **For Widget Extension** (if you want widgets):
   - File ‚Üí New ‚Üí Target
   - Select "Widget Extension"
   - Name: "UpcomingEventsWidgetExtension"
   - Copy `widgets/UpcomingEventsWidget.swift` to the extension target

3. **For CloudKit** (if you want iCloud sync):
   - Select app target ‚Üí Signing & Capabilities
   - Click "+ Capability"
   - Add "CloudKit"
   - Configure CloudKit container

4. **For App Group** (needed for Widget + CloudKit):
   - Select app target ‚Üí Signing & Capabilities
   - Click "+ Capability"
   - Add "App Groups"
   - Create: `group.com.yourcompany.birthdayreminder`
   - Repeat for Widget Extension target

## Step 4: Build and Run

### Option A: Build from Xcode
1. Select a simulator or connected device
2. Press ‚åòR or click Run button
3. Wait for build to complete

### Option B: Build from Terminal (Recommended)
```bash
# Start Expo dev server
npm run dev

# In another terminal, run iOS
npx expo run:ios
```

## Important Notes

### ‚ö†Ô∏è Don't Edit Generated Files Directly

The `ios/` folder is generated by Expo. If you need to customize:

1. **Use Expo Config Plugins** (preferred)
   - Add plugins to `app.json`
   - They modify native code during prebuild

2. **Use Expo Development Build**
   - Create custom development build
   - Add native code
   - Rebuild when needed

3. **Eject to Bare Workflow** (advanced)
   - Full control over native code
   - More maintenance required

### üîÑ Regenerating iOS Project

If you make changes to `app.json` or add new plugins:

```bash
npx expo prebuild --clean --platform ios
```

This regenerates the iOS project with new configurations.

## Troubleshooting

### "No such module" errors
- Run `cd ios && pod install`
- Or run `npx expo prebuild` again

### Build fails
- Check Xcode version (need 15+ for Expo SDK 54)
- Verify signing & capabilities
- Clean build folder: Product ‚Üí Clean Build Folder

### Can't find workspace
- Make sure to open `.xcworkspace`, not `.xcodeproj`
- Run `npx expo prebuild` first
